Parte 3 — Docker na EC2
Passo 1 — Criar a EC2 e instalar o Docker
01

Criar a EC2 e instalar o Docker via User Data

Uma nova instância dedicada para rodar containers

💡

O que é Docker? Uma plataforma que empacota uma aplicação e todas as suas dependências em um container — um ambiente isolado e portátil que roda igual em qualquer máquina.

  • Acesse o EC2 e execute uma nova instância com as mesmas configurações anteriores, com o nome docker-server
  • Na seção Firewall (grupos de segurança), selecione para permitir tráfego via: [X] SSH, [X] HTTP, [X] HTTPS:
  • No final da página, expanda Detalhes avançados e localize o campo Dados do usuário
  • Cole o script abaixo: será executado automaticamente na primeira inicialização:
#!/bin/bash
# Atualizar pacotes
apt update -y
apt upgrade -y

# Instalar o Docker e o Docker Compose
apt install -y docker.io docker-compose

# Iniciar o serviço e habilitar no boot
systemctl start docker
systemctl enable docker

# Adicionar o usuário ubuntu ao grupo docker
usermod -aG docker ubuntu
  • Clique em Executar instância
  • Aguarde as 3/3 verificações e conecte via Conexão de Instância do EC2
  • Confirme que o Docker foi instalado automaticamente:
$docker --version
# Saída esperada: Docker version 24.x.x, build ...

$docker compose version
# Saída esperada: Docker Compose version v2.x.x
💡

O que é User Data? Um script shell que a EC2 executa automaticamente na primeira inicialização, como root. Ideal para automatizar a configuração do servidor — sem precisar entrar na máquina para instalar nada manualmente.

✅

Docker e Docker Compose instalados automaticamente via User Data. A instância está pronta para rodar containers.

Passo 2 — Criar a imagem e publicar no Docker Hub
02

Build local, push para o Docker Hub e rodar com Compose

Construir a imagem, publicar no Docker Hub e subir com Docker Compose

⚠️

Contexto da atividade: este passo constrói uma aplicação de exemplo do zero para demonstrar o fluxo completo de build e push. Na prática, vocês já terão a aplicação pronta no GitHub — o processo será apenas clonar o repositório, buildar a imagem e fazer o push para o Docker Hub.

  • Conecte na docker-server via Conexão de Instância do EC2 e crie a pasta do projeto:
$mkdir meu-site && cd meu-site
  • Crie o arquivo HTML:
$nano index.html
<!-- index.html -->
<!DOCTYPE html>
<html>
<head><meta charset="UTF-8"><title>AppService</title></head>
<body style="font-family:sans-serif;text-align:center;padding:4rem">
  <h1>Minha aplicação rodando na AWS!</h1>
  <p>Container nginx · Docker Hub · ECS</p>
</body>
</html>
  • Salve (Ctrl+O → Enter → Ctrl+X) e crie o Dockerfile:
$nano Dockerfile
# Dockerfile
FROM nginx:alpine
RUN rm -rf /usr/share/nginx/html/*
COPY index.html /usr/share/nginx/html/index.html
EXPOSE 80
  • Salve e crie o docker-compose.yml:
$nano docker-compose.yml
# docker-compose.yml
services:
  web:
    build: .
    ports:
      - "80:80"
    restart: always
  • Suba o container localmente para testar:
$docker compose up -d --build
$docker compose ps
✅

Acesse http://<IP-PÚBLICO-DA-EC2> e confirme que a página carrega antes de continuar.

Publicar a imagem no Docker Hub

💡

O que é o Docker Hub? Um repositório público de imagens Docker. Ao fazer push da sua imagem, ela fica disponível para ser baixada de qualquer lugar — inclusive pelo ECS na AWS.

Etapa 1 — Criar a conta (obrigatório via navegador — feito uma única vez)

⚠️

Não tem conta no Docker Hub? O cadastro não pode ser feito pela linha de comando — é necessário acessar o site. Abra o navegador e acesse https://app.docker.com/signup, crie a conta e anote seu nome de usuário — ele será usado em todos os comandos a seguir. Quem já tem conta pode pular direto para a Etapa 2.

Etapa 2 — Criar o repositório no Docker Hub (via terminal, na EC2)

  • Antes de rodar o comando, você precisa de dois dados da sua conta. Usuário: é o nome que aparece no canto superior direito quando você acessa hub.docker.com — não é o e-mail, é o nome curto como joaosilva.
  • Token de acesso: é uma senha gerada pelo Docker Hub para uso em comandos e scripts. Siga os passos abaixo para criar o seu:

Como gerar o Access Token

  • Acesse hub.docker.com e faça login
  • Clique no seu avatar (canto superior direito) → Account Settings
  • No menu lateral, clique em Personal Access Tokens
  • Clique em Generate new token
  • Dê um nome descritivo, ex: ec2-lab
  • Em Access permissions, selecione Read & Write
  • Clique em Generate e copie o token imediatamente — ele só aparece uma vez
⚠️

Guarde o token em um lugar seguro agora. Se fechar a janela sem copiar, precisará gerar um novo.

  • Com usuário e token em mãos, execute os dois comandos abaixo no terminal da EC2 — substitua SEU_USUARIO e SEU_TOKEN pelos seus dados:

1. Obter o token JWT de autenticação

# Guarda o token JWT em uma variável de ambiente
$TOKEN=$(curl -s https://hub.docker.com/v2/auth/token \
  -H "Content-Type: application/json" \
  -d '{"identifier":"SEU_USUARIO","secret":"SEU_TOKEN"}' \
  | jq -r '.access_token')
💡

O que acontece aqui? A API do Docker Hub troca seu usuário e token por um JWT (JSON Web Token) — uma credencial temporária usada no próximo passo. O resultado fica salvo na variável $TOKEN e nunca aparece no terminal.

2. Criar o repositório usando o JWT

$curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"name":"appservice-demo","is_private":false}' \
  https://hub.docker.com/v2/repositories/SEU_USUARIO/
💡

O que esse comando faz? Cria o repositório público appservice-demo na sua conta usando o JWT obtido no passo anterior. Na resposta JSON você verá "name": "appservice-demo" confirmando a criação.

Etapa 3 — Login, tag e push (tudo no terminal)

  • Faça login no Docker Hub direto do terminal da EC2 — substitua SEU_USUARIO e SEU_TOKEN pelos seus dados:
$echo "SEU_TOKEN" | docker login -u SEU_USUARIO --password-stdin
# Login Succeeded
⚠️

Por que não usar só docker login? Sem parâmetros, o Docker tenta abrir um navegador para autenticação — o que não funciona em um terminal de servidor. O --password-stdin passa o token diretamente pela linha de comando, sem abrir browser e sem deixar a senha exposta no histórico do shell.

  • Crie a tag da imagem com seu usuário do Docker Hub:
# Formato: docker tag <imagem-local> <usuario>/<repositorio>:<tag>
$docker tag meu-site-web seu-usuario/appservice-demo:latest
⚠️

Como descobrir o nome da imagem local? Rode docker images — o nome aparece na coluna REPOSITORY. O Docker Compose normalmente cria como nomepasta-servico, ex: meu-site-web.

  • Envie a imagem para o Docker Hub:
$docker push seu-usuario/appservice-demo:latest
# Aguarde o upload de todas as camadas
🎉

Imagem publicada! Acesse https://hub.docker.com/r/seu-usuario/appservice-demo para confirmar. Agora ela pode ser usada em qualquer ambiente — inclusive no ECS.

💡

Comandos úteis do Docker Compose:
docker compose up -d — sobe em background
docker compose down — para e remove os containers
docker compose ps — lista o status
docker compose logs -f — acompanha os logs em tempo real
docker images — lista todas as imagens locais

Conceito — Usar uma imagem já publicada no Docker Hub
💡

Por que isso importa? Até agora o fluxo foi: código → build → push. Mas quando a imagem já existe no Docker Hub — seja a sua ou a de outra pessoa — não é preciso ter o código fonte nem fazer build. Basta fazer docker pull e rodar. É exatamente assim que o ECS vai usar sua imagem.

★

Baixar e rodar uma imagem direto do Docker Hub

Sem Dockerfile, sem build — apenas pull e run

  • Antes de subir o container, certifique-se de que nenhum container anterior está ocupando a porta 80:
# Para e remove todos os containers em execução
$docker stop $(docker ps -q)
$docker rm $(docker ps -aq)
⚠️

Por que fazer isso? Se você rodou containers nas etapas anteriores (ex: via Docker Compose), eles podem estar ocupando a porta 80. Tentar subir um novo container na mesma porta causa erro port is already allocated. Esses comandos garantem que a porta esteja livre antes de continuar.

  • Baixe a imagem do Hub para a máquina local (EC2 ou qualquer outra):
# Formato: docker pull <usuario>/<repositorio>:<tag>
$docker pull seu-usuario/appservice-demo:latest
# O Docker baixa cada camada da imagem automaticamente
  • Confirme que a imagem foi baixada:
$docker images
# Você verá a imagem listada com seu tamanho e data de criação
  • Rode o container diretamente a partir da imagem:
# -d: background -p: mapeia porta host:container --name: nome do container
$docker run -d -p 80:80 --name meu-container seu-usuario/appservice-demo:latest
  • Verifique que o container está rodando:
$docker ps
# STATUS deve aparecer como "Up X seconds"
✅

Acesse http://<IP-PÚBLICO-DA-EC2> e a aplicação estará no ar — sem nenhum arquivo local, apenas a imagem baixada do Hub.

💡

pull vs run: o docker run já faz o pull automaticamente se a imagem não existir localmente — ou seja, você pode pular o docker pull e usar apenas docker run diretamente. O docker pull explícito é útil quando você quer pré-baixar a imagem antes de rodar, ou garantir que tem a versão mais recente de uma tag existente.

⚠️

Para encerrar o container:
docker stop meu-container — para o container (pode ser reiniciado)
docker rm meu-container — remove o container (a imagem continua local)
docker rmi seu-usuario/appservice-demo:latest — remove a imagem local

💡

Relação com o ECS: o ECS faz exatamente isso — mas de forma automática e gerenciada. Quando você configura uma Task Definition apontando para seu-usuario/appservice-demo:latest, o ECS faz o pull da imagem e roda o container nas instâncias do cluster. Você não precisa fazer nada manualmente.

Parte 4 — ECS com imagem do Docker Hub
💡

O que é o ECS? Elastic Container Service — o orquestrador de containers da AWS. Em vez de gerenciar o Docker manualmente na EC2, o ECS faz isso por você: decide onde rodar o container, reinicia se cair e escala conforme necessário. Nesta parte vamos carregar a imagem própria dos alunos direto do Docker Hub.

Passo 3 — Criar o Cluster ECS
03

Criar o Cluster ECS (Fargate)

O agrupamento lógico que vai hospedar os containers

💡

O que é um Cluster? Um espaço lógico onde o ECS organiza e executa os containers. Com Fargate, não existe nenhuma EC2 para configurar ou gerenciar — a AWS provisiona a infraestrutura automaticamente nos bastidores. Você só define o container e o ECS cuida do resto.

  • No Console AWS, vá em ECS → Clusters → Criar cluster
  • Nome do cluster: mycluster
  • Em Infraestrutura, selecione Somente Fargate — desmarque EC2 se estiver selecionado
  • Deixe as demais opções no padrão
  • Clique em Criar — o cluster fica pronto em segundos
✅

Cluster criado. Nenhuma EC2, nenhum par de chaves, nenhum Auto Scaling Group — o Fargate gerencia tudo isso por baixo dos panos.

Passo 4 — Criar a Task Definition com imagem do Docker Hub
04

Criar a Task Definition

Apontar para a imagem do Docker Hub e configurar o container

💡

O que é uma Task Definition? A especificação do container — qual imagem usar, quanta CPU e memória alocar, quais portas expor. Pense nela como o docker run do ECS, só que declarativo.

  • Vá em ECS → Task Definitions → Criar nova task definition
  • Nome: minhatarefa
  • Tipo de inicialização: AWS Fargate
  • Sistema operacional: Linux/X86_64
  • CPU: 0.25 vCPU — Memória: 0.5 GB
  • Função da tarefa: selecione LabRole
  • Função de execução da tarefa: selecione LabRole
  • Em Container, preencha:
CampoValor
Nome do containerweb
Imagemseu-usuario/appservice-demo:latest
Mapeamento de porta — Container80
ProtocoloTCP
💡

Formato da imagem: use exatamente o mesmo nome que você deu no docker push — ex: seu-usuario/appservice-demo:latest. Como o repositório é público no Docker Hub, o ECS baixa automaticamente sem precisar configurar credenciais.

  • Clique em Criar
Passo 5 — Criar o Service e acessar o container
05

Criar o Service e acessar o container

Subir a imagem no cluster Fargate e acessar pelo navegador

💡

Task vs Service: uma Task é uma execução pontual do container. Um Service garante que um número definido de tasks fique sempre rodando — se uma cair, o ECS sobe outra automaticamente.

  • Vá em ECS → Clusters → mycluster → Services → Criar
  • Tipo de inicialização: Fargate
  • Em Detalhes do serviço, em Família da definição de tarefa: selecione minhatarefa
  • Nome do service: myservice
  • Expanda Redes, configure:
  • Em Grupo de segurança: crie um novo com nome ecs-sg, regra de entrada HTTP porta 80 origem 0.0.0.0/0
  • Clique em Criar
  • Aguarde até a faixa azul ficar verde, e o campo última implantação ficar Concluído — pode levar 1 a 2 minutos
  • Clique na aba Tarefas → clique no id da tarefa em execução → clique na aba Associações de rede, copie o Link externo
  • Acesse no navegador: http://<IP-PÚBLICO-DA-TASK>
🎉

Container rodando via ECS Fargate! A imagem do Docker Hub está sendo servida sem nenhuma EC2 para gerenciar — sem SSH, sem par de chaves, sem agente ECS.

💡

Diferença prática — EC2 direto vs ECS Fargate:
Com Docker na EC2 você instala, configura e gerencia tudo manualmente. Com ECS Fargate, você só descreve o container — a AWS cuida da infraestrutura, do reinício em caso de falha e da escalabilidade.
