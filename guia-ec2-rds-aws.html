O que vamos fazer

Criar EC2 + RDS, liberar o SG do RDS para a EC2 (porta 5432), instalar um servidor Node.js na EC2 e liberar o SG da EC2 para a internet (porta 3000).

Parte 1 — EC2 + RDS PostgreSQL
Passo 1 de 5
01

Criar a Instância EC2

O servidor que vai se conectar ao banco de dados

💡

O que é EC2? É uma máquina virtual na nuvem. Pense nela como um computador que você aluga na AWS — você acessa via terminal (SSH).

  • No Console AWS, vá em EC2 → Instâncias → Executar instâncias
  • No campo Nome, digite server1 — isso cria automaticamente a tag Name = server1
  • Em Início rápido, selecione Ubuntu
  • Em Imagem (AMI), selecione Ubuntu Server 24.04 LTS
  • Tipo de instância: t3.small
  • Par de chaves: selecione vockey
  • Em Configurações de rede, anote o nome da VPC padrão selecionada — você vai precisar escolher a mesma VPC na criação do RDS
  • No Firewall (Grupos de Segurança), selecione Criar Grupo de Segurança e marque as seguintes regras de entrada: [X] Permitir tráfego SSH, [X] Permitir tráfego HTTP e [X] Permitir Tráfego HTTPS
  • Configurar armazenamento: altere para 20 GiB
  • Clique em Executar instância
🏷️

O que são tags? São rótulos chave-valor anexados a recursos AWS. A tag Name é especial — é o nome exibido no console. Boa prática: sempre taguear seus recursos desde o início.

💡

Por que a VPC importa? EC2 e RDS só conseguem se comunicar se estiverem na mesma VPC. A zona de disponibilidade (AZ) não precisa ser a mesma — o que importa é a VPC. No lab, normalmente existe apenas uma VPC padrão, então basta confirmar que ambos estão nela.

⚠️

Ambiente de lab: o IP público da sua máquina muda a cada sessão do AWS Academy, então 0.0.0.0/0 no SSH evita ter que editar o Security Group toda vez. Em produção, sempre restrinja a porta 22 ao seu IP.

🧪 Teste — Conexão via EC2 Instance Connect

  • Aguarde o status da instância ficar ✔ 2/2 (ou 3/3) verificações aprovadas
  • Selecione a instância server1 e clique em Conectar
  • Na aba Conexão de instância do EC2, confirme que o usuário é ubuntu
  • Clique em Conectar — um terminal abre direto no navegador
✅

Conexão bem-sucedida — o prompt muda para ubuntu@ip-xxx-xxx-xxx-xxx:~$. Digite exit para sair.

Passo 2 de 5
02

Criar o Banco RDS PostgreSQL

O banco de dados gerenciado pela AWS

  • Vá em RDS → Bancos de dados → Criar banco de dados → Configuração completa
  • Mecanismo: PostgreSQL (versão padrão sugerida)
  • Método de criação: Criação fácil
  • Configuração: Sandbox
  • Identificador do banco: database-1
  • Usuário principal: postgres
  • Senha principal: anote! (exemplo: Ads-Cesar-2026)
  • Em Configurar conexão do EC2: selecione Conectar a um recurso de computação do EC2 → selecione sua instância
  • Clique em Criar banco de dados (leva ~5 min)
⚠️

Importante! Clique no banco → aba Segurança e conexão → clique em Endpoints. Em Endpoint e porta, salve o caminho. Guarde o endpoint, o usuário e a senha — você vai precisar na hora da conexão.

Passo 3 de 5
03

Instalar o cliente PostgreSQL na EC2

A ferramenta para se comunicar com o banco

  • Volte para a EC2 e conecte novamente no server1 via EC2 Instance Connect
  • Já dentro do Ubuntu, execute cada linha de comando abaixo:
# Atualizar pacotes
$sudo apt update -y
$sudo apt upgrade -y

# Instalar o cliente psql (PostgreSQL)
$sudo apt install -y postgresql-client
💡

O psql é o cliente de linha de comando do PostgreSQL. Com ele conseguimos nos conectar ao RDS e rodar comandos SQL diretamente do terminal.

Passo 4 de 5
04

Conectar no RDS PostgreSQL 🎯

EC2 falando com o banco

  • Primeiro vá no RDS, procurar o Endpoint: No RDS → Banco de dados → clique no id do seu banco → selecione Endpoints (em Conectar usando). Na seção Configurações adicionais → Endpoint e porta → copie o Endpoint
  • Execute o comando de conexão:
# Conectar no PostgreSQL via psql - sintaxe
$psql -h <ENDPOINT-DO-RDS> -U postgres -d postgres

# Exemplo real (substitua pelo seu endpoint):
$psql -h meu-banco.abc123.us-east-1.rds.amazonaws.com -U postgres -d postgres
  • Digite a senha cadastrada na criação do RDS
  • Se tudo deu certo, você verá o prompt: postgres=>
  • Teste alguns comandos SQL:
-- Criar um banco de teste
CREATE DATABASE mydb;

-- Conectar no novo banco
\c mydb

-- Criar uma tabela simples
CREATE TABLE alunos (id SERIAL, nome TEXT);

-- Inserir um registro
INSERT INTO alunos (nome) VALUES ('Maria');

-- Consultar
SELECT * FROM alunos;

-- Sair
\q
🎉

Conexão estabelecida! A EC2 está se comunicando com o RDS PostgreSQL através do Security Group configurado na porta 5432.

Parte 2 — Servidor Node.js na EC2
Passo 5 de 5
05

Instalar Node.js e rodar o servidor Hello World

Servidor HTTP acessível pela internet na porta 3000

🛡️

Lembra do Security Group? Assim como liberamos a porta 5432 no RDS para a EC2, agora precisamos liberar a porta 3000 no SG da EC2 para que qualquer pessoa na internet possa acessar o servidor Node.

  • Vá em EC2 → Security Groups e selecione o SG da sua EC2
  • Clique em Editar regras de entrada → Adicionar regra
  • Nome: node-http-server | Descrição: node-http-server
CampoValorPor quê?
TipoTCP personalizadoPorta customizada da nossa aplicação
Porta3000Porta padrão que o Node.js vai escutar
Origem0.0.0.0/0Qualquer IP pode acessar (internet pública)
  • Clique em Salvar regras
⚠️

Atenção: 0.0.0.0/0 libera para todo o mundo — OK para demonstração e testes. Em produção, restrinja ao seu IP ou use um Load Balancer.

  • Conecte na EC2 via EC2 Instance Connect e execute:
$sudo apt install curl -y

# Instalar o Node.js via NVM (gerenciador de versões)
$curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Recarregar o terminal para o nvm ficar disponível
$source ~/.bashrc

# Instalar a versão LTS do Node
$nvm install --lts

# Verificar se instalou corretamente
$node -v
$npm -v
💡

O que é NVM? Node Version Manager — facilita instalar e trocar versões do Node. É a forma recomendada de instalar Node em servidores Linux.

  • Crie a pasta do projeto e o arquivo do servidor:
# Criar pasta do projeto
$mkdir node-server && cd node-server

# Criar o arquivo server.js
$nano server.js
  • Cole o código abaixo no editor (Ctrl+V), salve com Ctrl+O → Enter, saia com Ctrl+X:
// server.js — servidor HTTP básico com Node.js puro
const http = require('http');
const PORT = 3000;

const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Hello World da EC2! ');
});

server.listen(PORT, () => {
  console.log(`Servidor rodando na porta ${PORT}`);
});
  • Inicie o servidor:
$node server.js
# ou em background: node server.js &
# Saída esperada: Servidor rodando na porta 3000
🎉

Servidor no ar! Vá na EC2, copie o IP público da instância e acesse no navegador:
http://<IP-PÚBLICO-DA-EC2>:3000
Você deve ver: Hello World da EC2!

💡

Opcional — Rodar em background: instale e configure o pm2 para manter o servidor vivo mesmo após fechar o SSH.

Problemas Comuns
⚠️

Connection timed out / sem resposta no psql
O Security Group do RDS não tem a regra na porta 5432 ou a origem está errada. Revise o Passo 3.

⚠️

FATAL: password authentication failed
Senha digitada incorreta. Tente redefinir no console: RDS → Modificar → Nova senha.

⚠️

psql: command not found
O cliente não foi instalado. Volte ao Passo 4 e rode sudo apt install -y postgresql-client.

⚠️

EC2 e RDS em VPCs diferentes
Não conseguem se comunicar por padrão. Ambos precisam estar na mesma VPC.

⚠️

Navegador não abre http://IP:3000 (ERR_CONNECTION_REFUSED)
Verifique se o servidor Node está rodando e se a porta 3000 está liberada no SG da EC2. Revise o Passo 6.

⚠️

nvm: command not found após instalação
Execute source ~/.bashrc para recarregar o terminal, ou feche e abra a sessão SSH novamente.
